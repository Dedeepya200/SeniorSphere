# SeniorSphere Role-Wise Documentation

## Senior Citizen

### Purpose
The senior role is the main end-user experience for community participation, wellbeing, and support.

### Main Capabilities
- sign up and join a community
- request a new community if needed
- view senior dashboard
- complete daily check-in
- read and create community posts
- view announcements
- create and join events
- offer or request skills
- create help requests
- send and receive direct messages
- receive notifications
- manage emergency contacts
- enable family visibility

### Typical Senior Flow
1. Sign up as `Senior Citizen`
2. Join a community
3. Log in and open dashboard
4. Check in daily
5. Participate in events, skills, posts, and help requests
6. Maintain emergency contacts and profile settings

## Family Member

### Purpose
The family role gives visibility into a senior’s wellbeing and activity without exposing full admin/moderator controls.

### Main Capabilities
- sign up as `Family Member`
- connect to seniors who allow family access
- view connected seniors list
- view a selected senior’s:
  - check-in status
  - help requests
  - joined events
  - requested skills
- raise help requests on behalf of a senior
- send reminders to a senior
- receive notifications related to senior activity

### Typical Family Flow
1. Sign up as `Family Member`
2. Search for and connect to a senior
3. Open the selected senior in the family dashboard
4. Review wellbeing and activity
5. Raise help or send reminder if needed

## Community Moderator

### Purpose
The moderator role manages one community’s content, requests, and member activity.

### Main Capabilities
- access moderator dashboard
- view community member count
- review pending help requests
- assign volunteers
- resolve help requests
- post community announcements
- review flagged posts
- remove posts, events, and skills
- monitor community content and participation

### Typical Moderator Flow
1. Log in with moderator credentials
2. Open moderator dashboard
3. Review incoming help requests
4. Assign support where needed
5. Review flagged posts
6. Remove unsafe or outdated content
7. Post announcements to the community

## Administrator

### Purpose
The admin role manages platform growth, moderation operations, and community lifecycle actions.

### Main Capabilities
- access admin dashboard
- view platform totals
- review pending community requests
- approve or reject community requests
- create and reassign moderators
- create replacement moderators
- view all communities and assigned moderators
- delete communities
- delete community-assigned accounts as part of community deletion

### Typical Admin Flow
1. Log in as `admin@seniorsphere.com`
2. Open admin dashboard
3. Review pending community demand
4. Approve, reject, or manage communities
5. Reassign moderators where required
6. Delete communities when necessary

## Shared Features Across Roles

Available to multiple roles depending on access rules:
- multilingual UI
- translation tools
- read-aloud support
- voice input
- notifications
- responsive mobile interface
- realtime updates
